## Overview
The `cypress/support` directory is a foundational part of our Cypress testing framework. It contains configuration files, utility functions, and custom commands that are made globally available to all our tests. Its primary role is to extend Cypress's default behavior, abstract complex or repetitive actions (like authentication), and provide a centralized location for shared testing logic. This ensures our tests remain DRY (Don't Repeat Yourself), readable, and maintainable across both End-to-End (E2E) and Component testing workflows.

## Usage
The files within this directory are automatically processed by Cypress before your test files are executed. You generally do not need to import them directly in your spec files, as Cypress handles the loading based on the test type.

*   **For E2E Tests**: Cypress automatically loads `e2e.ts` before running any E2E spec files. This file, in turn, imports `commands.ts` and other necessary utilities for the E2E suite.
*   **For Component Tests**: Cypress automatically loads `component.ts` before running any component spec files. This file handles the specific setup for component testing, including importing commands.
*   **Custom Commands**: Once a command is registered in `commands.ts` and imported via the entry files (`e2e.ts` or `component.ts`), it becomes available on the global `cy` object and can be used in any test file (e.g., `cy.login()`).

## Useful details
This directory is structured to separate concerns and maintain clarity. Below is a breakdown of its key components:

*   `auth-provider-commands/`: This sub-directory contains specialized, reusable commands for handling authentication flows with various third-party identity providers like Auth0, AWS Cognito, and Okta. This modular approach keeps the main `commands.ts` file clean and makes it easy to manage provider-specific logic.

*   `commands.ts`: This is the primary file for defining and registering custom Cypress commands. It acts as a central hub, often importing command modules from other files (like those in `auth-provider-commands`) and making them available to all tests via `Cypress.Commands.add()`.

*   `component.ts`: This file serves as the entry point for Cypress Component Testing support. It's used to import global styles, register component-specific commands, and configure the mounting process for components under test.

*   `e2e.ts`: This is the main support file that is loaded before any End-to-End test specs are run. It is responsible for importing global configurations and commands, ensuring the E2E testing environment is properly set up.

*   `utils.ts`: A collection of general-purpose helper functions that are not Cypress commands but are used within them or within test specs. This can include functions for generating test data, parsing values, or other reusable logic that supports the tests.

*   `component-index.html`: A custom HTML template used by the Cypress Component Test runner. It provides the base document where your components are mounted, allowing you to include global CSS, fonts, or meta tags necessary for consistent component rendering during tests.
