## Overview
The `support` directory is a fundamental part of the testing architecture in our software project, primarily dedicated to extending and customizing the default Cypress testing framework functionalities. This folder plays an essential role by housing various utilities, command extensions, and configurations that enhance the testing process. It provides a centralized structure for handling test-specific logic, custom commands, authentication mechanisms, component integration, and utility functions necessary for comprehensive end-to-end tests.

## Usage
To utilize this folder in your project, integrate its components into your Cypress testing setup. Each file and subdirectory serves a specific purpose in enhancing the test capabilities:
- **Custom Commands:** Extend Cypress functionalities by importing and using the custom commands defined in `commands.ts.md`.
- **Component Testing:** Utilize `component.ts.md` and `component-index.html.md` for integrating and testing individual UI components effectively.
- **End-to-End Testing:** Leverage `e2e.ts.md` for full-cycle testing scenarios to ensure application robustness.
- **Utility Functions:** Use `utils.ts.md` for shared utility functions that simplify repetitive testing tasks.
- **Authentication Support:** Configure and automate authentication workflows using scripts found in the `auth-provider-commands` directory, targeting specific providers like Auth0, AWS Cognito, and Okta.
