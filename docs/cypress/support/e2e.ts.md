This setup file globally modifies Cypress test behavior. Before each test, it intercepts all API calls to localhost:3001, stripping the 'if-none-match' header to force fresh responses, preventing caching. If tests run in a mobile context (detected by isMobile), API responses are throttled to 1 Mbps to mimic slow network conditions. No fixtures or test suites are defined here, only global intercept/middleware logic. Key commands: cy.intercept, res.setThrottle. Keywords: Cypress, intercept, throttle, caching, mobile, API, middleware, beforeEach, e2e, network simulation.
