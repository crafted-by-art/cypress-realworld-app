## Overview

The `support` folder in the Cypress project serves as the core extension layer for customizing and enhancing the behavior of your Cypress test suite. It provides a centralized location for reusable utility functions, custom commands, test setup logic, and authentication helpers—which are all critical for scalable, maintainable, and expressive end-to-end (E2E) and component testing.

This folder plays a fundamental role by:

- Defining and registering Cypress custom commands (`commands.ts`), enabling you to create expressive, application-specific testing APIs.
- Providing utilities (`utils.ts`) to DRY (Don't Repeat Yourself) common test logic.
- Managing both global and component-specific setup/configuration (`e2e.ts`, `component.ts`), ensuring your test environment is consistently prepared.
- Organizing authentication helper scripts in the `auth-provider-commands` subfolder, making it easy to authenticate against third-party identity providers across all tests.
- Supplying support content for Cypress component testing (`component-index.html`), facilitating proper mounting and rendering of components in test isolation.

By leveraging the contents of this directory, test writing becomes faster, cleaner, and more robust, while ensuring best practices are shared across the team.

## Usage

To utilize the code in the `support` folder within your Cypress project, follow these standard steps:

1. **Automatic Loading:**  
   Cypress automatically loads `support/e2e.ts` before each test file for E2E tests, and `support/component.ts` for component tests. This means your commands, setups, and utilities are globally available across your test files.

2. **Registering Custom Commands:**  
   - Add functions to `support/commands.ts` to define new `Cypress.Commands.add(...)` helpers.
   - Use these helpers in your tests as `cy.yourCustomCommand(...)`.

3. **Utilizing Utilities:**  
   - Import methods from `support/utils.ts` in your custom commands or directly in test specs for reusable logic.

4. **Authentication Helpers:**  
   - The `auth-provider-commands` directory (`auth0.ts`, `cognito.ts`, `okta.ts`) provides specialized login commands for different authentication providers. Call these helpers in your tests or in setup steps to automate logging into your application:
     ```js
     cy.loginWithAuth0();
     cy.loginWithCognito();
     cy.loginWithOkta();
     ```
   - See `auth-provider-commands/README_2025-07-14_17-47-49.md` for provider-specific setup and usage.

5. **Component Testing:**  
   - `component-index.html` and `component.ts` help ensure proper environment setup for component tests, so your UI components mount and render correctly.

6. **Extending Support:**  
   - You can add more commands, utilities, or provider scripts, and organize them in subfolders for further modularity.

## Useful details

- **File Structure Purpose**
  - `commands.ts`: Houses project-wide reusable Cypress commands. Keeps your test specs concise and expressive.
  - `utils.ts`: Contains helper functions and utilities to support commands and tests.
  - `e2e.ts`: Handles E2E-specific support code and initialization.
  - `component.ts`: Handles component test-specific setup and customizations.
  - `component-index.html`: Supplies a base HTML scaffold for mounting components in tests, mirroring app structure.
  - `auth-provider-commands/`: Modularizes authentication logic, reducing duplication and complexity in your tests.

- **How to Extend**
  - Add new `.ts` modules or folders for additional providers or utilities.
  - Register new commands in `commands.ts` to encapsulate common user flows or assertions.

- **Configuration**
  - No manual import is required in test files; setup files in this folder are automatically loaded by Cypress according to the test type.
  - You can configure Cypress to use this folder for both E2E and component testing in your `cypress.config.js`.

- **Documentation**
  - Each script (e.g., `auth-provider-commands/README_2025-07-14_17-47-49.md`) provides more granular documentation for specific modules—refer to these as you implement related features.

- **Maintaining Best Practices**
  - Regularly refactor and organize helpers, utilities, and commands to keep your test suite scalable and clean.

In summary, the `support` folder is the foundation of Cypress test customization for your project. All shared, reusable, or bootstrapping logic for tests should live here to ensure reliability and ease of use across your automation suite.
