## Overview
This directory contains the entire automated testing suite for the project, built using the Cypress framework. Its primary purpose is to ensure the quality, reliability, and stability of the application by providing comprehensive end-to-end (E2E), API, and component testing.

As a cornerstone of our development lifecycle and quality assurance strategy, this test suite is integrated into our CI/CD pipeline. It automatically validates new features, prevents regressions, and provides developers with rapid feedback on their changes, ensuring that the application meets our high standards before reaching production.

The folder is structured to separate configuration, reusable support code, and the test specifications themselves, promoting maintainability and scalability of our testing efforts.

## Usage
To execute the tests, you will need Node.js and npm/yarn installed.

1.  **Install Dependencies:**
    Navigate to the project's root directory and run the installation command:
    ```bash
    npm install
    ```

2.  **Run Tests Interactively:**
    To open the Cypress Test Runner, which allows you to watch tests run in real-time and use time-traveling debug tools, execute:
    ```bash
    npx cypress open
    ```

3.  **Run Tests Headlessly:**
    To run all tests in the terminal (ideal for CI/CD environments), use the following command:
    ```bash
    npx cypress run
    ```
    You can also target specific tests by using the `--spec` flag:
    ```bash
    npx cypress run --spec "cypress/tests/ui/auth.spec.ts"
    ```

## Useful details
The structure of the `cypress` directory is organized to promote clarity and code reuse. Below is a breakdown of its key components:

| Component | Purpose |
| :--- | :--- |
| **`cypress.config.ts`** | The main configuration file for Cypress. It defines global settings such as the base URL, viewport dimensions, environment variables, and test timeouts. This is the first place to look when setting up the test environment. |
| **`/support`** | Contains reusable code and setups that are accessible across all test files. This includes custom commands (e.g., `cy.login()`), global hooks, and utility functions. Centralizing this logic keeps our test files clean and DRY (Don't Repeat Yourself). See its [README](./support/README_2025-07-14_16-39-50.md) for more. |
| **`/tests`** | The core directory holding all test specification (`.spec.ts`) files. It is further subdivided to separate different types of tests, such as API, UI, and authentication provider tests. See its [README](./tests/README_2025-07-14_16-41-48.md) for more details on its subdirectories. |
| **`global.d.ts`** | A TypeScript declaration file. It provides type definitions for custom Cypress commands created in the `support/commands.ts` file. This enables IntelliSense and type-checking in modern code editors, improving the developer experience. |
