## Overview
This `demo` folder contains a collection of example and proof-of-concept Cypress tests. Its primary purpose is to provide a practical demonstration of specific testing features and to serve as a starting point for rapid test development. The tests within this directory, such as `cypress-studio.spec.ts`, are pre-configured with necessary setup steps like database seeding and user authentication, creating an ideal environment for developers to quickly build out new tests, particularly using visual tools like Cypress Studio. In the context of the larger project, this folder acts as a living-documentation and sandbox for our E2E testing strategies.

## Usage
The tests in this folder are designed to be used interactively, especially with the Cypress Test Runner. The primary use case involves leveraging Cypress Studio to accelerate test creation.

To utilize the tests in this folder:
1.  Ensure your Cypress configuration (`cypress.config.ts` or `cypress.json`) has the `experimentalStudio` flag set to `true`.
2.  Open the Cypress Test Runner.
3.  Navigate to and run a test from the `demo` directory, for example `cypress-studio.spec.ts`.
4.  Once the test setup (e.g., login) is complete, use the Cypress Studio "Add Commands to Test" feature in the Test Runner to begin recording your interactions with the application UI.
5.  After recording a workflow, save the generated commands as a new test. This allows you to build a robust test case without writing the code manually.

## Useful details
- **Not for Production Suite:** The tests in this directory are primarily for demonstration and development purposes. They may not be part of the main CI/CD regression pipeline and should be considered illustrative examples.
- **Dependency on Cypress Studio:** The key value of files like `cypress-studio.spec.ts` is their integration with Cypress Studio. This feature must be enabled in the project's Cypress configuration for the intended workflow.
- **Pre-configured State:** The tests rely on a specific initial state established by `beforeEach` hooks (e.g., database seeding, user login). Modifying this setup may impact the ability to record new tests successfully.
- **Onboarding Tool:** This folder is an excellent resource for new developers joining the team to understand our testing setup and quickly get started with writing E2E tests.
