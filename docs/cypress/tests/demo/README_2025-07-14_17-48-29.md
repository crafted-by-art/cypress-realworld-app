## Overview

The `demo` folder is a part of the Cypress end-to-end (E2E) testing suite. Its primary purpose is to demonstrate the setup and basic structure required for writing and executing UI-driven tests using Cypress Studio. This folder plays a foundational role in automating the testing process, ensuring that the application's database is seeded, users are authenticated, and the environment is consistent before each test scenario. The included test suite serves as a template for creating and managing interactive tests, focusing on workflows such as creating new transactions and bank accounts in a web application.

## Usage

To utilize the tests within this folder:

1. **Prepare the project:**  
   Ensure you have Cypress installed and your project configured for running Cypress tests.

2. **Access the demo suite:**  
   The main file in this folder is `cypress-studio.spec.ts.md`, which outlines how to set up and prepare the testing environment. This file can be expanded to include step definitions using Cypress Studio's record-and-playback features.

3. **Test environment preparation:**  
   Each test within this folder will:
   - Seed the database via `cy.task("db:seed")` to reset application data.
   - Find a user using `cy.database("find", "users")`.
   - Log in the user through `cy.login()` to simulate authenticated user interactions.

4. **Running the tests:**  
   Launch Cypress using the command line (typically `npx cypress open` or `npx cypress run`), then select the `cypress-studio.spec.ts.md` demo test suite to execute or modify it.

5. **Extending the demo:**  
   Use Cypress Studio to visually record new test steps for UI interactions such as creating transactions or bank accounts. Insert generated steps into the placeholder areas in the spec file.

## Useful details

- **Template nature:**  
  The provided spec file is a scaffold—key pre-test actions (database seed, user login) are implemented, but test steps are intentionally left empty and are expected to be filled in using Cypress Studio.

- **Database management:**  
  By seeding the database before every test, you ensure test repeatability and isolation.

- **Authentication flow:**  
  The login command (`cy.login()`) simulates authentic user sessions, allowing for testing of authenticated features.

- **No external fixtures:**  
  The test suite currently does not load or interact with external data fixtures.

- **Recommended usage:**  
  Use this folder as a starting point for writing your own E2E tests. It is particularly useful in onboarding new contributors or when first adopting Cypress and Cypress Studio in your project.
