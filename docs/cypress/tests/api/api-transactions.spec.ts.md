Test suite "Transactions API" validates the Transactions REST API. It seeds the DB, authenticates a user, and prepares test data before each test. It checks fetching user transactions, pending requests (including by date range), transactions between contacts (with pagination), and public transactions. It tests creating payments and requests via POST, including checking statuses and response fields. PATCH requests are tested for updating transaction status and error handling for invalid fields. Key commands: cy.request, cy.database, cy.task, cy.loginByApi. Keywords: API, transactions, payment, request, Cypress, PATCH, POST, GET, user, contacts, e2e.
