## Overview
This directory contains all API-level end-to-end (E2E) tests for the application, written using the Cypress framework. The primary purpose of these tests is to validate the backend API endpoints directly, bypassing the user interface. This ensures the reliability, correctness, and stability of the server-side logic for core application features.

These tests are a critical component of our Continuous Integration (CI) process. By verifying the API contract, they help catch regressions and validate new features at a layer that is faster and more focused than full UI-based E2E tests. The suites cover a wide range of functionalities, including user management, banking operations, social interactions, and data handling.

## Usage
To execute these tests, ensure that the project dependencies are installed and the backend application server is running and accessible. The tests can be run collectively or individually using standard Cypress commands.

**Prerequisites:**
1.  Install project dependencies: `npm install`
2.  Ensure the backend application is running.

**Running all API tests:**
```bash
npx cypress run --spec "cypress/tests/api/*.spec.ts"
```

**Running a specific test file (e.g., user tests):**
```bash
npx cypress run --spec "cypress/tests/api/api-users.spec.ts"
```

## Useful details
The tests in this directory are organized by API resource, with each file corresponding to a specific set of related endpoints. This modular structure makes it easy to locate, run, and maintain tests for a particular feature.

### Component Breakdown

*   **`api-users.spec.ts`**: Validates user management endpoints, including user creation, authentication (login), retrieval, search, and updates.
*   **`api-bankaccounts.spec.ts`**: Ensures that authenticated users can correctly perform Create, Read, and Delete (CRD) operations on their bank accounts via RESTful and GraphQL APIs.
*   **`api-transactions.spec.ts`**: Covers the core financial transaction endpoints, including creating payments/requests, and viewing personal, contact, and public transactions.
*   **`api-banktransfers.spec.ts`**: Verifies the `/bankTransfers` endpoint, ensuring users can securely retrieve their bank transfer history.
*   **`api-contacts.spec.ts`**: Tests the API for managing a user's contact list, covering creation, retrieval, and deletion of contacts.
*   **`api-comments.spec.ts`**: Validates the functionality for creating and fetching comments associated with transactions.
*   **`api-likes.spec.ts`**: Ensures the `/likes` endpoint functions correctly for creating and retrieving likes on transactions.
*   **`api-notifications.spec.ts`**: Tests the full lifecycle of notifications, including retrieval, creation, and updates via the `/notifications` endpoint.
*   **`api-testdata.spec.ts`**: A utility test suite that validates the `/testData/:entity` endpoint, which is crucial for seeding the application with mock data during automated test runs.
