Test suite "Likes API" verifies GET and POST endpoints for likes on transactions. Before each test, the database is seeded, a user is authenticated via API, and a like’s transactionId is loaded. One test checks if GET /likes/:transactionId returns a 200 status and exactly 1 like. Another test validates that POST /likes/:transactionId creates a like and returns status 200. Key commands: cy.request, cy.task, cy.database, cy.loginByApi. Keywords: likes, API, transactionId, authentication, GET, POST, e2e, seeding, Cypress, assertions. No external fixtures are loaded.
