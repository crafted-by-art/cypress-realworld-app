## Overview
This directory serves as the central repository for all end-to-end (E2E) tests for the project, implemented using the Cypress framework. Its primary purpose is to provide comprehensive test coverage across the entire application stack, from the backend API to the final user interface.

In the broader project architecture, this `tests` folder is a cornerstone of our quality assurance strategy and CI/CD pipeline. These automated tests are executed regularly to prevent regressions, validate new features, and maintain overall application integrity, ensuring a high-quality product is delivered. The tests are logically categorized into distinct subdirectories based on their focus: direct API testing, core UI functionality, authentication flows, and demonstrations.

## Usage
The tests within this directory are designed to be run using the Cypress test runner. Developers can execute them either interactively for debugging or headlessly as part of an automated process.

**To open the interactive Cypress Test Runner:**
```bash
npx cypress open
```

**To run all tests headlessly (ideal for CI environments):**
```bash
npx cypress run
```

**To run a specific suite of tests, target a subdirectory:**
```bash
# Example: Run only the API tests
npx cypress run --spec "cypress/tests/api/**/*.spec.ts"
```

## Useful details
The test suite is organized into the following subdirectories, each serving a distinct purpose in our testing strategy. This structure helps in locating, maintaining, and adding new tests efficiently.

*   **`./api`**
    *   **Purpose**: Contains E2E tests that interact directly with the application's backend API endpoints. These tests bypass the UI to validate server-side logic, data integrity, and API contract adherence. They are critical for ensuring the stability of the core business logic.
    *   For more information, see the [API Tests Documentation](./api/README_2025-07-14_16-40-14.md).

*   **`./ui`**
    *   **Purpose**: Holds E2E tests that simulate user interactions with the core application's user interface after a user is authenticated. These tests cover primary user workflows and essential features like creating transactions, viewing data, and managing settings.
    *   For more information, see the [UI Tests Documentation](./ui/README_2025-07-14_16-41-26.md).

*   **`./ui-auth-providers`**
    *   **Purpose**: Focuses specifically on testing the complete user authentication and onboarding journeys via various third-party identity providers (e.g., Auth0, Google, Okta). These tests validate the critical path from login to initial application access.
    *   For more information, see the [Auth Provider Tests Documentation](./ui-auth-providers/README_2025-07-14_16-40-37.md).

*   **`./demo`**
    *   **Purpose**: A collection of example and proof-of-concept tests. This folder is used to demonstrate specific Cypress features or to prototype new testing strategies in a pre-configured environment. It serves as a valuable resource for learning and experimentation.
    *   For more information, see the [Demo Tests Documentation](./demo/README_2025-07-14_16-41-06.md).
