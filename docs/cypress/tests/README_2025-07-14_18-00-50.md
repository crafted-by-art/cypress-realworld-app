## Overview
This directory serves as the root for the entire Cypress automated testing suite. It provides a comprehensive, multi-layered approach to quality assurance, encompassing everything from direct API validation to full end-to-end (E2E) user interface testing.

The primary purpose of this folder is to organize and house all tests required to validate the application's stability, functionality, and reliability. The structure is designed to separate concerns, allowing for targeted test runs and easier maintenance. The subdirectories are organized as follows:

-   **`api/`**: Contains integration tests that directly target the application's backend API endpoints. These tests ensure data integrity, business logic, and contract adherence without involving the UI.
-   **`ui/`**: Holds E2E tests that simulate user interactions with the core application's user interface after they have logged in.
-   **`ui-auth-providers/`**: A specialized set of E2E tests focused exclusively on validating the critical authentication and user onboarding flows via third-party identity providers (e.g., Google, Okta, Auth0).
-   **`demo/`**: Includes example tests designed to showcase tooling features like Cypress Studio, serving as an educational resource for the development team.

Together, these test suites form a robust safety net that verifies the application's correctness from the server to the browser.

## Usage
These tests are designed to be run both locally during development and automatically in a Continuous Integration (CI) environment.

### Running Tests
To run all tests headlessly (ideal for CI):
```bash
npx cypress run
```

To open the interactive Cypress Test Runner for local development and debugging:
```bash
npx cypress open
```

### Running Specific Test Suites
You can execute tests from a specific subdirectory by targeting it with the `--spec` flag. This is useful for focusing on a particular area of the application.

**Run only API tests:**
```bash
npx cypress run --spec "cypress/tests/api/**/*.spec.ts"
```

**Run only core UI tests:**
```bash
npx cypress run --spec "cypress/tests/ui/**/*.spec.ts"
```

### Adding New Tests
When creating new tests, place them in the appropriate subdirectory based on their function:
-   New API endpoint validation goes into `api/`.
-   New tests for user-facing features (post-login) go into `ui/`.
-   Tests for a new third-party login provider go into `ui-auth-providers/`.

## Useful details
-   **Multi-Layered Strategy**: This test suite deliberately separates API and UI tests. API tests are generally faster and more stable, providing a quick check of the backend's health. UI tests are more comprehensive and ensure the user experience is intact.
-   **Start with the Demo**: If you are new to the project or Cypress, the `demo/` folder is an excellent starting point. It contains practical examples and demonstrates how to leverage tools like Cypress Studio to accelerate test creation.
-   **Detailed Documentation**: For more specific information about the purpose and scope of the tests within each sub-category, please refer to the `README.md` file located within each respective subdirectory (`api/`, `ui/`, etc.).
