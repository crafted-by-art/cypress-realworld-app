## Overview
The `tests` folder is a comprehensive repository of automated testing scripts within the software project. It is structured into multiple subfolders, each tailored to specific types of tests, including UI tests, API tests, demo interaction tests, and authentication provider tests. This folder is integral to the project as it facilitates the execution of end-to-end and integration tests, ensuring the robustness and reliability of both the frontend and backend components of the application.

## Usage
To utilize the testing scripts within this folder, follow these general steps:
1. **Setup Cypress:** Ensure Cypress is installed in your project by running `npm install cypress` in the terminal.
2. **Open Cypress:** Use the command `npx cypress open` to launch the Cypress Test Runner which helps in organizing and executing tests.
3. **Select Tests:** Navigate through the Cypress Test Runner UI to select the relevant tests from any desired subfolder (e.g., `api`, `ui`, `demo`, `ui-auth-providers`) based on the functionalities you wish to validate.
4. **Execute Tests:** Click on the selected test scripts to execute them. This action will initiate automated testing scenarios that will run against your application.
5. **Review Results:** After tests execution, analyze the results and logs provided by Cypress to ensure the functionalities are working as expected.

For higher efficiency, consider setting up a CI/CD pipeline using Cypress CLI commands and integrate these tests for automated nightly or pre-release test runs.

  - `api`: Contains test specifications for API endpoints covering bank accounts, transactions, notifications, users, and likes. These scripts verify data integrity and communication reliability between client and server.

  - `ui-auth-providers`: Focuses on different authentication providers, ensuring secure and efficient login and user sessions with services like Auth0, Cognito, Google, and Okta.
