Test suite covers notification features. It tests that after switching to User C and visiting notifications, a new payment request from User A appears in the list. A second test ensures an empty notifications state by intercepting the API with an empty response, logging in, and confirming the UI shows “No Notifications.” Key commands: `cy.switchUserByXstate`, `cy.loginByXstate`, `cy.getBySelLike`, `cy.intercept`, assertions on notification content and state, and visual snapshots at multiple steps. Keywords: notifications, requests, Cypress, assertions, intercept, UI state, multi-user, empty state, snapshots, selectors.
