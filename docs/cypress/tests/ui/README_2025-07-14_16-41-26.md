## Overview
This directory contains a comprehensive suite of Cypress end-to-end (E2E) tests designed to validate the core user interface (UI) and functionality of the application. The primary purpose of these tests is to simulate real user interactions with key features after they have successfully logged in.

In the context of the larger project, this folder is critical for ensuring the stability and reliability of the main user-facing application flows. It covers everything from managing bank accounts and transactions to updating user settings and viewing notifications. By automating these tests, we can rapidly detect regressions, validate new feature development, and maintain a high-quality user experience across the application's core feature set.

## Usage
These E2E tests are designed to be run against a running instance of the application. Ensure the development server is active and the application is accessible at the `baseUrl` configured in your `cypress.config.ts`.

To execute all the tests within this UI suite, you can run the following command from the project root:

```bash
# Run all UI tests in headless mode
npx cypress run --spec 'cypress/tests/ui/**/*.spec.ts'

# Or open the Cypress Test Runner to run tests interactively
npx cypress open
```

Prerequisites for running these tests include having a properly configured environment, which may involve setting specific environment variables (`CYPRESS_*`) for API endpoints, user credentials, or other configuration details.

## Useful details
This section provides a breakdown of each test file within this directory and its specific purpose.

### Component Breakdown

*   **`auth.spec.ts`**
    *   **Purpose:** Validates the fundamental authentication flows of the application. It covers user sign-up, login, logout, and session management to ensure the core access control mechanisms are robust and functional.

*   **`bankaccounts.spec.ts`**
    *   **Purpose:** Focuses on the bank account management feature. These tests verify that users can successfully create, view, and delete their bank accounts, ensuring the reliability of this crucial setup step.

*   **`new-transaction.spec.ts`**
    *   **Purpose:** Covers the entire "New Transaction" user flow. It simulates creating, sending, and requesting payments between users, validating form inputs, API interactions, and UI updates for both parties involved.

*   **`notifications.spec.ts`**
    *   **Purpose:** Ensures the notification system works as expected. It tests that notifications are correctly generated and displayed for various user activities, such as receiving payments, likes, and comments.

*   **`transaction-feeds.spec.ts`**
    *   **Purpose:** Tests the different transaction feeds ("Public," "Friends," and "Mine"). It verifies correct rendering, pagination, filtering by date/amount, and responsive design of the transaction lists.

*   **`transaction-view.spec.ts`**
    *   **Purpose:** Validates the detailed view of a single transaction. These tests confirm that a user can see transaction details and interact with them, including liking, commenting, and responding to payment requests.

*   **`user-settings.spec.ts`**
    *   **Purpose:** Tests the "User Settings" page. It ensures that users can view their profile information, make changes, and successfully save the updates, validating both the form UI and the backend integration.
