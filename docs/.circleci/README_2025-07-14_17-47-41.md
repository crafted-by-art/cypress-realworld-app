## Overview

The `.circleci` folder contains configuration files that define the Continuous Integration (CI) pipeline for the project using [CircleCI](https://circleci.com/). The primary purpose of this folder is to automate essential development processes such as dependency installation, code linting, building the project, running unit and end-to-end (E2E) tests, and managing code coverage reporting.

Within a larger software project, the `.circleci` configuration ensures code quality and stability by automatically running tests and static analysis for every code change, providing fast feedback to developers and supporting a robust DevOps workflow.

## Usage

To make use of the CircleCI configuration provided in this folder, follow these steps:

1. **Connect Your Repository to CircleCI:**  
   Ensure your GitHub or other version control repository is linked with CircleCI.

2. **Add the `.circleci/config.yml`:**  
   The configuration file (referenced here as `config.yml.md`) should be renamed/copied to `.circleci/config.yml` at the root of your repository to take effect. CircleCI automatically detects this file and uses it to orchestrate your pipeline.

3. **Pipeline Automation:**  
   Once configured, every push or pull request to your repository will trigger the defined jobs:
   - Install dependencies
   - Lint code for errors and styling issues
   - Build the project
   - Run server-side unit tests
   - Execute Cypress component, API, and UI (E2E) tests across multiple browsers (Chrome and Firefox) and viewports (desktop and mobile)
   - Generate and upload code coverage reports to Codecov
   - Store test artifacts
   - Perform visual regression tests using Percy

4. **Customizing:**  
   You can modify the `config.yml` to adapt jobs, commands, or environment settings to fit your project's unique requirements.

## Useful details

- **Parallelization:**  
  The configuration leverages CircleCI's parallelism feature to speed up Cypress testing across different environments and viewports.

- **Multi-browser & Responsive Testing:**  
  Cypress tests are run on both Chrome and Firefox, and both desktop and mobile viewports, ensuring browser and device compatibility.

- **Artifact Management:**  
  Test artifacts (logs, screenshots, etc.) are stored for review of failures or debugging.

- **Visual Testing:**  
  Integration with Percy enables automated visual regression testing to catch unintended UI changes.

- **Code Quality Gate:**  
  Automated linting and testing act as a quality gate for all code merged into the repository.

- **Coverage Reporting:**  
  Code coverage data is uploaded to Codecov, enabling visible test coverage metrics for the project.
