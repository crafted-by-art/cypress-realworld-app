## Overview
This directory contains the configuration files for our CircleCI continuous integration (CI) pipeline. Its primary purpose is to automate the project's testing and validation processes, ensuring code quality and stability with every change pushed to the repository.

The configuration defined here orchestrates a sophisticated testing workflow built around Cypress. It is responsible for automatically executing our component, API, and UI test suites in a parallelized environment. This automation is crucial for providing rapid feedback to developers, catching regressions early, and maintaining a high standard of quality before code is merged or deployed.

## Usage
The CI pipeline is automatically triggered and requires no manual instantiation. When a developer pushes commits to a branch on our version control system (e.g., GitHub), CircleCI detects the `config.yml` file within this directory and executes the defined workflow.

Developers interact with this system primarily by:
1.  Pushing code to the repository.
2.  Monitoring the build status and results in the CircleCI dashboard.
3.  Reviewing detailed test reports, visual diffs from Percy, and code coverage reports from Codecov, which are linked directly from the build results.

To validate changes to the CI configuration itself before pushing, you can use the CircleCI CLI locally:
```bash
# Install the CircleCI CLI (if not already installed)
# ...

# Navigate to the project root and validate the config
circleci config validate
```

## Useful details
*   **Parallel Execution:** The workflow is heavily optimized for speed. It runs multiple Cypress jobs in parallel, significantly reducing the total time required to get feedback on a set of changes.
*   **Comprehensive Test Matrix:** Tests are executed across multiple configurations to ensure broad compatibility. This includes running suites on both **Chrome** and **Firefox** browsers, and for both **desktop** and **mobile** viewports.
*   **Visual Regression Testing:** We leverage **Percy** integration (`percy exec ...`) to capture UI snapshots during test runs. This automatically detects and highlights any unintended visual changes, preventing UI bugs from reaching production.
*   **Code Coverage:** The pipeline integrates with **Codecov** to upload test coverage reports. This allows us to track how much of our codebase is covered by automated tests and enforce coverage standards on new contributions.
